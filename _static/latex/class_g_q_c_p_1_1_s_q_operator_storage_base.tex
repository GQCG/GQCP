\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename \+\_\+\+Matrix\+Representation, typename \+\_\+\+Vectorizer, typename \+\_\+\+Derived\+Operator$>$\newline
class G\+Q\+C\+P\+::\+S\+Q\+Operator\+Storage\+Base$<$ \+\_\+\+Matrix\+Representation, \+\_\+\+Vectorizer, \+\_\+\+Derived\+Operator $>$}

A type that takes care of storing the matrix elements of one-\/ and two-\/electron operators and their components.

\begin{DoxyNote}{Note}
The sole reason why this base class is implemented, is to allow S\+Q\+Operators that don\textquotesingle{}t have to support vector space arithmetic, like {\ttfamily \mbox{\hyperlink{class_g_q_c_p_1_1_evaluatable_scalar_r_s_q_one_electron_operator}{Evaluatable\+Scalar\+R\+S\+Q\+One\+Electron\+Operator}}}.
\end{DoxyNote}

\begin{DoxyTemplParams}{Template Parameters}
{\em \+\_\+\+Matrix\+Representation} & The type used to represent the set of parameters/matrix elements/integrals for one component of a second-\/quantized operator. \\
\hline
{\em \+\_\+\+Vectorizer} & The type of the vectorizer that relates a one-\/dimensional storage of matrix representations to the tensor structure of the second-\/quantized operators. This allows for a distinction between scalar operators (such as the kinetic energy or Coulomb operator), vector operators (such as the spin operator) and matrix/tensor operators (such as quadrupole and multipole operators). \\
\hline
{\em \+\_\+\+Derived\+Operator} & The type of the operator that derives from this class, enabling \mbox{\hyperlink{class_g_q_c_p_1_1_c_r_t_p}{C\+R\+TP}} and compile-\/time polymorphism. \\
\hline
\end{DoxyTemplParams}
